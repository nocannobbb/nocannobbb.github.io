如何编写c++版本的线程池

借鉴于 https://www.cnblogs.com/lzpong/p/6397997.html

在编写线程池之前，我们首先需要了解为什么需要线程池？
在传统的c++多线程中，多线程的编写都是使用默认的方法，也就是创建，执行，销毁这个过程。我们可以想象，使用默认的方法，在线程执行时间远大于创建与销毁时间时，会比较合理，因为创建销毁这个过程所占用的时间不多。 但相反的来说，若每个线程执行的时间很短，而应用在短时间内拥有较大并发量时，线程会不断地进行创建与销毁，占用大量的时间，这个时候，传统的线程管理方法就不再适用, 我们就需要使用线程池来进行线程的管理。

当然也可以是用来练习c++

一、c++基础知识

匿名函数(lamda)

capture->return-type{function-body}

  1. [capture]: 以什么样的形式接收参数
    [&]         //截取外部作用域中所有变量，并作为引用在函数体中使用
[=]         //截取外部作用域中所有变量，并拷贝一份在函数体中使用
[=, &foo]   //截取外部作用域中所有变量，并拷贝一份在函数体中使用，但是对foo变量使用引用
[bar]       //截取bar变量并且拷贝一份在函数体重使用，同时不截取其他变量
[this]      //截取当前类中的this指针。如果已经使用了&或者=就默认添加此选项。

[x, &y] //x 按值捕获, y 按引用捕获
[&, x] //x显式地按值捕获. 其它变量按引用捕获
[=, &z] //z按引用捕获. 其它变量按值捕获```

  1. (parameters): 匿名函数的具体传参

    如果没有传参，圆括号也可以省略，变成[capture]->return-type{function-body}

  2. return-type: 匿名函数的返回值类型

    如果返回值为void类型，或者函数里面只有一个返回语句，则return-type可以省略，变为capture{function-body}

  3. eg：

    [](int& x) { ++x; } // 没有return语句 -> lambda 函数的返回类型是'void' 
[]() { ++global_x; } // 没有参数,仅访问某个全局变量 
[]{ ++global_x; } // 与上一个相同,省略了()
[](int x, int y) -> int { int z = x + y; return z; }```
